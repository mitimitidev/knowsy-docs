MVP build roadmap — sequenced milestones

M0~1–2 wk

Foundation & scaffold

Bun + Next.js 16 app, Drizzle/Neon schema (user, property, report, order), Better Auth, Stripe, Inngest wired, preview-per-PR CI.

Grounded in 2.1 stack · 2.2 data model · 2.3 infra
M1~2 wk

Core happy path — one free report

Address validation → resolve property → checkout → durable workflow (keyed by reportId) pulling 1–2 free sources (FEMA NFHL, NCES) → render a basic report. Proves the end-to-end spine.

Grounded in 3.3 pipeline · 5.1 search/checkout wireframes
M2~2–3 wk

Data breadth via the connector pattern

Add hazard/GIS, weather, places, news connectors behind the uniform adapter; source_fetch cache-and-reuse + TTLs; partial report states.

Grounded in 3.1 vendor stack · 3.3 connector pattern
M3~2–3 wk

Deed extraction + review

Deed ingestion (upload/county first), Claude schema extraction (OCR fallback), confidence scoring, review_item queue + manual-review UI.

Grounded in 3.2 OCR/extraction · 4.1 per-report cost
M4~2 wk

Summary, report UX & observability

Claude report summary, full report-detail experience (loading → partial → success), Sentry + Inngest dashboards, cost-per-report instrumentation.

Grounded in 5.2 report wireframes · 2.3 observability · 4.x cost KPI
M5~1–2 wk

Hardening & launch

Per-vendor rate-limit tuning, PII encryption + retention policy, alerting, load test against vendor ceilings, sign-off.

Grounded in 2.3 trade-offs · 6.1 risk register

Future phases (post-MVP)

  • Premium deed tier — DataTree: add the authoritative title-grade / deed-image source once volume justifies the ~$2,500/mo floor (4.1) — gate behind a premium report tier.
  • Gated / partnership sources: Reddit, Nextdoor, First Street, Zillow/MLS — each needs an approval or contract cycle (3.1); start the conversations during MVP, integrate as they clear.
  • Geographic expansion: generalize beyond Westchester — per-county GIS adapters and the multi-state property vendors already abstract behind the connector pattern.
  • Scale-out triggers: if a single flow needs independent scaling/runtime, extract it from the modular monolith (the 2.3 split trigger); revisit Temporal only if orchestration complexity outgrows serverless.
  • Analytics & trust: historical trends, comps, and surfacing per-field provenance/confidence to build user trust in the report.

How the roadmap ties to Discovery findings

  • Architecture (Phase 2): M0–M1 stand up the stateless serverless spine + durable workflow; the modular-monolith decision sets the M5 / future extract triggers.
  • Data pipeline (Phase 3): the connector pattern lets M2 add breadth without rework; M3 slots deed extraction into the same durable steps.
  • Costs (Phase 4): free/transparent vendors first (M1–M2), the expensive DataTree contract deferred to a future premium tier; cost-per-report instrumented at M4 to drive tier decisions.
  • Product/UX (Phase 5): wireframes feed M1 (search/checkout) and M4 (report detail) directly.
