Scaling considerations

  • Scales per report (variable): Claude tokens, OCR pages, property-records calls, Google Places/Address calls — these dominate at volume and are the levers to optimize.
  • Scales gently: Vercel compute/bandwidth, Inngest runs, Neon compute — autoscale absorbs load; serverless keeps idle cost near zero.
  • Storage grows monotonically: Vercel Blob accumulates deed files — apply a retention policy (2.2) and serve derived/thumbnail assets, not raw scans, to control egress.
  • The real ceiling is vendors, not infra (2.3): cache-and-reuse keyed by property is the primary cost lever — re-purchases within TTL skip paid fetches entirely.
  • Cost-control levers: prompt caching (Claude), field masks (Places), aggressive TTLs on stable data (deeds, flood), and free public sources for hazard/schools/news.

MVP-friendly infrastructure recommendations

  • Stay serverless + scale-to-zero: Vercel + Neon + Inngest free/low tiers keep fixed cost ~$60–150/mo until traffic justifies upgrades.
  • Defer the enterprise contract: launch without DataTree (county/WRO + user upload for deeds); add it only when deed volume or a premium tier pays for the ~$2,500/mo floor.
  • Start Realie at $50/mo and let overage/tier track actual volume; reassess vs RealEstateAPI ($599/mo) only if comps/skip-tracing enter scope.
  • Lean on free + transparent vendors (FEMA, GIS, NCES, GDELT, Visual Crossing, Google) so the only contract-gated dependency is optional.
  • Instrument cost per report from day one (2.3 KPI) — it's the metric that tells you when to move tiers or renegotiate.
