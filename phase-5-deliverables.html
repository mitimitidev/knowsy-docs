Phase 5 · Week 3–4

Product Definition & UX Wireframes

Low-fidelity wireframes for the complete user journey: address search, checkout, report generation states, and the report detail experience.

2 tasks 1 wireframe set Weeks 3–4

User journey overview

Flow

Search → Confirm property → Checkout → Processing (async) → Report ready → Report detail

The user experience is intentionally minimal. The goal of v1 is validation — getting a paying user to a completed report. Polish and account management come post-MVP.

Screen inventory

Screen 1 — Address search

Address autocomplete + property card

Single input field with address autocomplete. On selection, a property card appears with the matched owner of record, parcel ID, and assessment details pulled from county data. User confirms this is the correct property before proceeding.

Key decisions: Confirm step is intentional — address ambiguity (unit numbers, similar street names) must be resolved before a report is created and a charge is made.

Screen 2 — Checkout

What's included + Stripe payment

Summary of what the report contains (chain of title, encumbrances, flags). Flat-fee pricing display. Stripe Elements embed for card entry. Email field for report delivery. No account creation required for v1.

Screen 3 — Processing states

Report generation progress

Post-payment screen with a live progress indicator. Steps displayed: retrieving deed → reading document → analyzing chain → building report. Estimated time shown (typically 1–3 minutes). Page polls a status endpoint every 5 seconds.

On completion, auto-redirects to the report detail page. On failure, shows a clear error state with a support contact option and an automatic refund trigger.

Screen 4 — Report detail Open wireframe

Full report view

The report detail wireframe is the most developed artifact from Phase 5. It covers the full structured report layout: property summary bar, chain of title timeline, encumbrances table, and AI-extracted red flags with severity indicators.

The wireframe also specifies annotation states for flagged deed fields — hover behavior, confidence scoring display, and the "review needed" indicator for low-confidence OCR extractions.

UX decisions logged

No user accounts (v1)

Reports delivered by email link. Access is time-limited (30 days). Removes auth friction from the purchase flow. User accounts are a planned v2 feature.

Auto-refund on failure

If the pipeline fails to produce a report, an automatic Stripe refund is issued. The user sees a clear error state. No manual support intervention required for the common failure case.

No report editing

The report is read-only. Users can flag items for their own notes but cannot modify the sourced data. Integrity of the underlying record is a key trust signal.

PDF export

A single-button PDF export is included in the report detail view. The PDF is pre-rendered server-side — not a browser print stylesheet — for consistent formatting.
